# DataTable 사용 규칙

리스트형 데이터 화면의 핵심. 정렬 / 필터 / 페이지네이션 / 빈 상태 / 로딩이 함께 놓이는
복합 컴포넌트라, **각 상태에 대한 책임을 명확히 분리** 하는 것이 9 점대 디자인 시스템의
조건이다.

## column 디자인

| 컬럼 종류 | 정렬 | 폭 | 정렬 가능? |
|----------|------|-----|----------|
| ID / 코드 | 좌측 정렬 | 80 ~ 120px 고정 | 보통 X (의미 없음) |
| 이름 / 제목 | 좌측 정렬 | flex / 가장 넓게 | O |
| 금액 / 카운트 | **우측 정렬** | 100 ~ 140px 고정 | O |
| 상태 (Badge) | 좌측 정렬 | 100 ~ 120px 고정 | O |
| 날짜 | 좌측 정렬 | 120px 고정 | O (기본 desc) |
| 액션 (Button) | 우측 정렬 | 80 ~ 120px 고정 | X |

`enableSorting` 을 켜면 헤더가 button 으로 wrapping 되어 키보드/스크린리더 접근성 자동
확보. 정렬 가능 헤더는 시각적으로 ArrowUpDown 아이콘으로 어포던스 부여.

## 상태 매트릭스 (필수 시나리오)

| 상태 | 무엇을 보여주는가 |
|------|------------------|
| **Loading** | 동일한 그리드 폭의 Skeleton row 4 ~ 5 줄 (검색/필터는 비활성) |
| **Empty (필터 결과 0)** | EmptyState (action: 필터 초기화) |
| **Empty (첫 방문)** | EmptyState (action: 생성 CTA) - 위와 다른 카피 |
| **Default** | 데이터 + Pagination |
| **Mobile** | 가로 스크롤 + 검색/필터 세로 스택 |

⚠ Loading 상태에서 헤더는 그대로 노출하되, 정렬 button 은 `disabled` 처리해 잘못된 토글
방지.

## Pagination 위치

- **테이블 하단 중앙** 이 표준
- Pagination + 합계 ("123 개 중 1 ~ 10") 를 한 줄에 둘 때는 **합계 좌측 / Pagination 우측**
- 페이지 이동 외에 page size 변경이 필요하면 좌측에 Combobox

## hover / selection

- row hover 는 `hover:bg-interactive-hover` (intent 토큰, 다른 컴포넌트 hover 와 통일). 너무 진한 색은 시각 피로
- multi-select 가 필요하면 첫 컬럼에 Checkbox + 헤더에 전체 선택 Checkbox

## 안티패턴

- 동일 데이터를 두 번 fetch 하는 패턴 (Loading 상태에서 데이터 비움 + 다시 fetch)
  → Skeleton 으로 레이아웃 유지하면서 invisible 한 데이터를 흐리게 보여주는 게 깜빡임 적음
- 컬럼 폭이 가변인데 헤더가 줄바꿈 → height 들쭉날쭉. 핵심 컬럼은 폭 고정
- Pagination 을 테이블 위 + 아래 양쪽에 → 사용자가 두 번 보고 헷갈림. 50 row 이상일 때만 위쪽 추가 검토
- 모바일에서 컬럼 강제 줄바꿈 → 가로 스크롤이 더 안전. 모바일 전용으로 column 일부 hide 도 옵션
- 빈 결과에 단순 "결과 없음" 텍스트만 → EmptyState 컴포넌트로 승격해 action 을 함께 제공
