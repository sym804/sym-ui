# EmptyState 사용 규칙

데이터가 없을 때 보이는 빈 영역의 표준. 사용자가 "에러 인가 / 내가 잘못한 건가 / 다음에
무엇을 해야 하나" 를 즉시 파악하도록 도와야 한다.

## 시나리오 분류 (가장 중요)

| 시나리오 | title 어조 | description | action |
|---------|-----------|-------------|--------|
| **첫 방문** (데이터가 한 번도 없음) | "첫 OO 을 만들어보세요" | 무엇이 가능한지 한 줄 | `primary` 버튼 1 개 (생성) |
| **빈 검색 결과** | `"키워드" 결과가 없습니다` | 철자 / 필터 조정 안내 | `outline` 1 ~ 2 개 (필터 초기화 / 새로 검색) |
| **권한 없음** | "접근 권한이 없습니다" | 어떻게 받을 수 있는지 | `outline` (관리자에 요청) |
| **오류 / 실패** | "불러오지 못했습니다" | 일시적인 문제일 가능성 | `outline` (다시 시도) |

## 카피 길이 가이드

- **title**: 한 문장, **20 자 이내**. 명사 종결보다 동사 종결이 행동 친화적
- **description**: **1 ~ 2 문장 / 60 자 이내**. 너무 자세하면 "오류 안내" 처럼 보임
- **action**: 최대 2 개. primary 1 + secondary 1 조합. action 3 개 이상은 **EmptyState
  를 잘못 쓴 신호** (Form 또는 Dialog 가 더 적절)

## icon 사용

- 명확히 의미가 있는 경우만 (예: 검색 결과 X → `Search` 아이콘, 잠긴 권한 → `Lock`)
- 단순히 빈 공간 채우기 위한 icon 은 오히려 노이즈

## 안티패턴

- description 이 80 자 넘는 장문 → FAQ 페이지 링크로 빠지는 게 낫다
- title 에 emoji 와 중복 의미 (예: "결과 없음 😢") → 시각/스크린리더 양쪽에 노이즈
- action 이 페이지 외부로 이동 (홈으로 돌아가기) → 사용자가 컨텍스트를 잃는다. 같은 페이지에서 해결 가능한 액션을 우선
- 다크 모드에서 dashed border 가 안 보임 → `border-border` 가 정확히 적용되어 있는지 확인
